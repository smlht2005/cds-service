## 先理解「為什麼需要 ValueSet」

在醫療系統裡，同一個概念可能有幾十種代碼。「結核病」光是 ICD-10 就有 A15、A16、A17、A18、A19 五個範圍，加上 Z16.34，如果每次都一個一個比對，程式會又長又容易遺漏。ValueSet 就是一個「代碼清單的名字」，你只要說「屬於這個清單嗎？」就好，不用列舉每一個代碼。

## 初學者最常問的三個問題

**Q1：valueset 宣告本身不做任何事，對嗎？**
完全正確。`valueset "...":'urn:oid:...'` 只是「登記一個別名」，就像在程式裡寫 `const TB_CODES = "..."`。真正的判斷邏輯在後面的 `define` 裡才發生。

**Q2：如果 PLACEHOLDER 沒填，會怎樣？**
CQL 引擎執行時會報錯「找不到 ValueSet」。這個 OID 必須向疾管署資訊室或你們醫院的 FHIR 主機管理員索取，填入後才能部署上線。

**Q3：能不能不用 valueset，直接在 define 裡寫代碼？**
可以，但不建議。直接寫的話大概長這樣：
```cql
where C.code.coding.code in { 'A15', 'A16', 'A17', 'A18', 'A19', 'Z16.34' }
```
這樣做的缺點是：未來疾管署新增代碼時，你要修改程式。用 valueset 的話，只要在 FHIR 伺服器更新清單內容，程式完全不用動。
